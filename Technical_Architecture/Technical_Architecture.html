Mimari özeti: Mobil uygulama tüm istekleri API Gateway üzerinden iletir. Gateway JWT doğrulaması ve rate limiting yapar. Backend servisleri birbirinden bağımsız çalışır — masal servisi Claude API'yi, ses servisi OpenAI TTS ve S3'ü, ödeme servisi RevenueCat'i yönetir. İş kuralı: Masal metni önce üretilip kullanıcıya gösterilir; ses dönüşümü yalnızca kullanıcı talep ettiğinde tetiklenir. Her masal maksimum 1.300 kelime (~10 dk ses) ile sınırlıdır.

Akış özeti: Kullanıcı formu doldurup gönderir → Backend kredi kontrolü yapar → Yaş bilgisiyle prompt oluşturur → Claude API masalı üretir (maks. 1.300 kelime) → Backend kelime sayısını doğrular → İçerik güvenlik filtresinden geçer → PostgreSQL'e kaydedilir → Metin olarak kullanıcıya gösterilir. Kullanıcı "Sesli dinle" butonuna basarsa TTS akışı ayrıca tetiklenir: S3'te cache yoksa OpenAI TTS (tts-1, onyx voice) çağrılır, ses S3'te saklanır ve 1 saatlik imzalı URL kullanıcıya döner. audioUrl DB'de saklanmaz — her istekte on-the-fly üretilir (BR-002).
